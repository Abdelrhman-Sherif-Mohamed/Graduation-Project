1. firebase-config.js
================================================================

الهدف:
  ملف مشترك بين كل الصفحات - بيهيئ Firebase ويصدّر auth و db

المكتبات المستخدمة:
  - initializeApp  : بيهيئ مشروع Firebase بالـ config
  - getAuth        : بيرجع كائن المصادقة (تسجيل الدخول)
  - getFirestore   : بيرجع كائن قاعدة البيانات

firebaseConfig:
  - بيانات المشروع من Firebase Console
  - هي بيانات عامة (public) وآمنة في الـ frontend
  - الحماية الحقيقية بتكون في Firestore Security Rules

الـ exports:
  - auth : بيتستخدم في كل صفحة للتحقق من تسجيل الدخول
  - db   : بيتستخدم للقراءة والكتابة في Firestore

Firebase بيُستخدم في المشروع لـ:
  1. Authentication : تسجيل الدخول / إنشاء حساب / إعادة كلمة المرور
  2. Firestore      : حفظ الحجوزات / بيانات المستخدمين / بيانات الأطباء

2. js/doctors-data.js
================================================================

الهدف:
  مصدر البيانات المركزي - بيعرّف كل التخصصات والأطباء
  بيتحمل قبل أي script تاني يحتاج البيانات

SPECIALTIES Array:
  - مصفوفة فيها 12 تخصص طبي
  - كل تخصص object فيه:
    * id      : معرف فريد (heart / dentist / bones ...)
                بيتستخدم في الـ URL (?doctor=ahmed) والفلترة
    * name    : اسم التخصص بالعربي
    * icon    : class أيقونة Bootstrap Icons
    * img     : صورة التخصص (URL من Unsplash)
    * desc    : وصف مختصر
    * doctors : array من الأطباء في هذا التخصص

  كل طبيب في doctors array فيه:
    * id    : معرف فريد (ahmed / sara / ...) - بيتستخدم في ?doctor=id
    * name  : اسم الطبيب
    * title : لقبه الطبي
    * exp   : خبرته
    * img   : صورته (URL)

getDoctorById(doctorId):
  - بتلف على كل التخصصات وتدور على الطبيب بالـ id
  - لو لقته: بترجع بياناته + اسم تخصصه + id تخصصه
  - لو مش موجود: بترجع null
  - بتستخدمها appointment.html لملء بيانات الطبيب
    لما تيجي من رابط ?doctor=ahmed مثلاً

  d-flex align-items-center justify-content-center : يمركز الكارت

--- الفورم ---
  input-group        : أيقونة على يمين الـ input
  togglePass         : زرار إظهار/إخفاء كلمة المرور
  forgotLink         : رابط "نسيت كلمة المرور"
  btnSpinner         : spinner يظهر أثناء التحقق

--- JavaScript ---
  signInWithEmailAndPassword : تسجيل الدخول بـ Firebase Auth

  منطق تحديد الـ role:
  1. بيجيب بيانات المستخدم من users/{uid} في Firestore
  2. لو مش موجود، بيبحث في doctors collection بالـ email
     (الأطباء بيتسجلوا بـ {doctorId}@alshifa-clinic.com)
  3. بيحفظ في users للمرة الجاية عشان يكون أسرع

  التوجيه بعد الدخول:
  - doctor → dashboard.html
  - admin  → create-doctors.html
  - patient → patient.html

  رسائل الخطأ:
  - auth/user-not-found    : البريد غير مسجّل
  - auth/wrong-password    : كلمة المرور غير صحيحة
  - auth/invalid-email     : صيغة البريد غير صحيحة
  - auth/too-many-requests : تجاوز عدد المحاولات
  - auth/invalid-credential: البريد أو كلمة المرور غير صحيحة

  sendPasswordResetEmail : بيبعت reset email عبر Firebase

الهدف: لوحة تحكم الطبيب - عرض وإدارة الحجوزات

--- الهيكل ---
  d-flex : Sidebar + المحتوى جنب بعض أفقياً

--- Sidebar ---
  - يعرض اسم الطبيب وتخصصه (من Firebase)

  - على الموبايل: position fixed + translateX(100%) يخفيه
  - .show class تجيبه للشاشة

--- Views (5 views) ---
  overview  : بانر التخصص + 4 إحصائيات + جدول اليوم المختصر
  today     : جدول حجوزات اليوم الكامل (مع واتساب وعنوان)
  all       : كل الحجوزات مع تابات فلترة
  patients  : قائمة المرضى (مريض واحد لكل patientId فريد)
  specialty : جدول الحجوزات مع أعمدة إضافية خاصة بالتخصص

--- SPEC_CONFIG ---
  object فيه إعدادات كل تخصص:
  - icon     : أيقونة Bootstrap Icons
  - color    : اللون الرئيسي
  - bg       : لون الخلفية الفاتح
  - desc     : وصف التخصص
  - extraCols: أعمدة إضافية في جدول التخصص
    مثلاً: heart → ['ضغط الدم', 'معدل القلب']

--- Firebase Auth ---
  onAuthStateChanged : بيراقب حالة تسجيل الدخول
  - مش مسجل → login.html
  - يجرب users/{uid} أولاً (أسرع - direct lookup)
  - لو مش موجود → يبحث في doctors بالـ email
  - لو مش طبيب → patient.html

--- loadData(doctorId) ---
  query(collection(db,'appointments'), where('doctorId','==',doctorId))
  بيجيب كل الحجوزات الخاصة بالطبيب
  بيرتبها من الأحدث للأقدم بـ createdAt.seconds

--- renderAll() ---
  بيبني كل الـ Views دفعة واحدة بعد تحميل البيانات
  Set() عشان نحسب المرضى الفريدين فقط

--- renderTable(tbodyId, appts, mode) ---
  mode 'short' : أعمدة مختصرة (Overview)
  mode 'full'  : أعمدة كاملة (Today View)
  mode 'all'   : أعمدة متوسطة (All View)

--- renderPatientsTable() ---
  Map() عشان يشيل التكرار (نفس المريض أكثر من حجز)
  يعرض آخر حجز لكل مريض

--- renderSpecialtyPanel() ---
  يبني mini-card في Overview
  يبني جدول كامل في View التخصص مع الأعمدة الإضافية

--- filterAll(status, el) ---
  فلترة client-side بدون طلب جديد من Firestore

--- switchView(viewName, el) ---
  يخفي كل الـ Views ويظهر المطلوب فقط
  يحدث العنوان في الـ Topbar

--- statusBadge(s) ---
  pending   → badge أصفر
  confirmed → badge أخضر
  cancelled → badge أحمر

--- actionBtns(id, status, prefix) ---
  pending   → زرار تأكيد + إلغاء
  confirmed → زرار إلغاء فقط
  cancelled → نص "ملغي" بدون أزرار

--- confirmAppt / cancelAppt ---
